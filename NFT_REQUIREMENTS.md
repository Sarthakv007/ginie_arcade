## 🎨 General NFT Specifications

### Image Requirements
- **Format:** PNG with transparency
- **Dimensions:** 512x512px
- **Color Depth:** 24-bit RGB + 8-bit Alpha
- **File Size:** < 500KB per image

### Design Guidelines
- **Common (Bronze):** Simple design, basic colors, minimal effects
- **Uncommon (Silver):** Enhanced details, metallic sheen, slight glow
- **Rare (Gold):** Rich details, golden accents, particle effects
- **Epic (Platinum):** Maximum detail, animated elements, premium effects
- **Legendary:** Unique design, multiple layers, special animations

### Metadata Structure
```json
{
  "name": "[Achievement Name]",
  "description": "[Achievement Description]",
  "image": "ipfs://[CID]/[filename].png",
  "attributes": [
    { "trait_type": "Game", "value": "[Game Name]" },
    { "trait_type": "Achievement Type", "value": "[Type]" },
    { "trait_type": "Rarity", "value": "[Common/Uncommon/Rare/Epic/Legendary]" },
    { "trait_type": "XP Bonus", "value": "[Amount]" },
    { "trait_type": "Mint Date", "value": "[Timestamp]" }
  ]
}
```

## 📝 Notes

1. **Progression System:** NFTs should follow natural game progression (easy → hard)
2. **XP Balance:** Higher rarity = more XP to incentivize rare achievements
3. **Visual Consistency:** Each game should have a cohesive art style
4. **Gas Optimization:** Batch mint when possible to reduce transaction costs
5. **Uniqueness:** Each NFT is unique to the player-achievement combination
6. **Tradability:** Consider making NFTs soul-bound (non-transferable) for true achievement proof
